# NL-to-SQL MCP Interface

![MCP Version](https://img.shields.io/badge/MCP-v0.21.2-blue)

A robust **Model Context Protocol (MCP)** server that bridges the gap between Large Language Models (LLMs) and local relational databases. This project enables AI agents such as Claude to autonomously explore database schemas and execute complex SQL queries using purely natural language.

---

# 📌 Project Overview

This project demonstrates an **agentic tool-calling architecture** where an LLM interacts with a local SQLite database through structured MCP tools.

Instead of manually writing SQL queries, the AI agent can:

- Discover available database tables
- Understand table schemas and relationships
- Construct complex `JOIN` queries
- Execute safe, read-only SQL statements
- Return insights in natural language

The system effectively transforms a local database into an AI-accessible knowledge source.

---

# 🎯 Real-World Use Case: MLOps Registry

To demonstrate practical utility, this repository includes a mock **MLOps Model Registry** database.

The registry tracks:

# 🏗️ Architecture

The server is built using the **FastMCP Python SDK** and exposes three specialized tools to the LLM client.

## Available MCP Tools

### 1. `list_tables()`

Returns all user-defined tables in the database.

#### Example Output

```python
["models", "datasets", "training_runs", "deployments"]
```

---

### 2. `describe_table(table_name)`

Returns schema information for a specific table, allowing the LLM to infer relationships and foreign keys.

#### Example

```python
describe_table("models")
```

#### Example Output

```python
[
  {"column": "id", "type": "INTEGER"},
  {"column": "model_name", "type": "TEXT"},
  {"column": "architecture", "type": "TEXT"}
]
```

---

### 3. `execute_query(query)`

Executes secure, read-only SQL queries generated by the LLM.

## Security Features

- Only `SELECT` statements are allowed
- Dangerous operations are blocked:
  - `DROP`
  - `DELETE`
  - `UPDATE`
  - `ALTER`
  - `INSERT`

#### Example

```python
execute_query("""
SELECT model_name, map_score
FROM training_runs
ORDER BY map_score DESC
LIMIT 1;
""")
```

# 🤖 Example Agentic Workflow

## User Prompt

```text
What was the highest mAP score achieved by a model deployed on NVIDIA Jetson, and what dataset was it trained on?
```

---

## Behind the Scenes

### Step 1 — Discover Database Tables

```python
list_tables()
```

### Step 2 — Understand Schemas

```python
describe_table("models")
describe_table("datasets")
describe_table("training_runs")
```

### Step 3 — Build Context

The agent infers:
- foreign key relationships
- relevant columns
- deployment constraints

### Step 4 — Generate SQL Query

```sql
SELECT m.model_name,
       d.dataset_name,
       t.map_score
FROM training_runs t
JOIN models m ON t.model_id = m.id
JOIN datasets d ON t.dataset_id = d.id
WHERE t.hardware = 'NVIDIA Jetson'
ORDER BY t.map_score DESC
LIMIT 1;
```

### Step 5 — Return Natural Language Response

```text
The highest mAP score achieved on NVIDIA Jetson was 92.4% by YOLOv8, trained on the COCO dataset.
```

# 🔒 Security Design

This MCP server is intentionally designed as a **read-only interface**.

## Restricted SQL Operations

The following commands are blocked:

- `DROP`
- `DELETE`
- `INSERT`
- `UPDATE`
- `ALTER`
- `TRUNCATE`

This prevents destructive database operations while still enabling analytical querying.
